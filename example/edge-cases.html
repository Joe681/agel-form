# 分散参数

在设计上，主张的是聚拢所有参数为一个 form 对象，组件会在创建时候自动注入全局属性和方法到该对象，副作用就是form 对象不可设置为计算属性，会导致响应式失效。

为了在某些场景下保证灵活性，可以使用 attach 进行分散参数，配置项与 form 一致，当该对象属性发生变化时候，会同步属性合并到 form 对象中。

 <agel-form v-model="form" :attach="{data,items,inline}"/>
点击查看代码 
<template>
  <div class="demo">
    <p>
      <span>行内表单：</span>
      <el-switch v-model="inline"></el-switch>
      <span style="margin-left:10px">form.data.name：</span>
      <el-input v-if="form.data" v-model="form.data.name" style="width:200px"></el-input>
    </p>
    <agel-form v-model="form" :attach="{data,items,inline}"> </agel-form>
  </div>
</template>
 
<script>
export default {
  data() {
    return {
      form: {},
      data: { name: "属性同步合并到 form.data" },
      inline: false,
    };
  },
  computed: {
    items() {
      return [
        { label: "姓名", prop: "name" },
        { label: "手机", prop: "phone" },
        { label: "地址", display: this.inline },
      ];
    },
  },
};
</script>

# 扩展&忽略 item 属性

当你想对 item 添加自定义的属性来实现某些需求，又不想该属性注入到 component 中，可以使用 item-ignore-keys 属性。

<agel-form v-model="form" :item-ignore-keys="['collapseAlive']"/>

# 动态属性的冲突

当组件的动态属性与现有的 item.xx 属性产生重名冲突，可使用 $component 属性解决。

 {
  label: "radio 组件",
  component: "el-radio",
  border: true,
  slots: "边框",
  $component: {
    label: "radio组件的label属性",
  },
},
